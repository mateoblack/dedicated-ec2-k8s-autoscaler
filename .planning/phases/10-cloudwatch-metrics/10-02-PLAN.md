<task type="auto">
  <name>Task 2: Create bash-metrics.ts module</name>
  <files>lib/scripts/bash-metrics.ts, lib/scripts/index.ts, test/scripts/bash-metrics.test.ts</files>
  <action>
Create bash-metrics.ts following bash-logging.ts pattern:

1. `getBashMetricsFunctions()` returns bash script string containing:
   - `emit_metric()` function that calls `aws cloudwatch put-metric-data`:
     - Parameters: metric_name, value, unit (default: Count)
     - Uses CLUSTER_NAME env var for namespace: `K8sCluster/${CLUSTER_NAME}`
     - Uses INSTANCE_ID env var for InstanceId dimension
     - Uses REGION env var for --region flag
     - Handles errors gracefully (logs warning, doesn't fail script)
   - `emit_metric_with_dimensions()` for custom dimensions
     - Parameters: metric_name, value, unit, dimensions_json
     - dimensions_json is "Name=Key,Value=Val Name=Key2,Value=Val2" format
   - Helper constants: `METRIC_COUNT="Count"`, `METRIC_MILLISECONDS="Milliseconds"`, `METRIC_BYTES="Bytes"`

2. Update lib/scripts/index.ts to export getBashMetricsFunctions

3. Create test/scripts/bash-metrics.test.ts:
   - Test getBashMetricsFunctions returns non-empty string
   - Test contains emit_metric function definition
   - Test contains emit_metric_with_dimensions function definition
   - Test contains aws cloudwatch put-metric-data command
   - Test contains proper dimension format (Name=,Value=)
   - Test contains CLUSTER_NAME usage for namespace
   - Test contains INSTANCE_ID usage for dimension
   - Test contains REGION usage for --region flag
   - Test module exports correctly from lib/scripts

IMPORTANT: emit_metric should not fail the script on error - use `|| log_warn "Failed to emit metric"` pattern. Metrics are observability, not critical path.
  </action>
  <verify>npm run test:code -- --testPathPattern=bash-metrics passes all tests</verify>
